内网信息收集

针对本机：收集网络情况和敏感信息

针对所处的网络：收集网段连通信息并进行内网资产探测，并且是否存在域环境

为了规避杀软与安全设备，尽量避免触发杀软警告的敏感操作于大量扫描，避免应急事件的发生

本机信息收集

IP/权限/系统版本/用户

进入到该主机之后，可以先探测该网络内的网段信息

IP

ipconfig /all

权限

系统版本

可以判断已拿到主机的系统详细信息

通过主机的名称来定位该主机的具体作用

以及可以查看该主机的补丁数来找到获取该主机的漏洞详情，进行进一步的利用

用户

进入该系统之后可以先查看该主机的用户信息

Windows

域环境中有许多常见的用户身份

可以利用net user /domain、net group /domain来获取域用户/域用户组的信息

域sid、用户sid，获取到这个可为之后票据的伪造提供了条件

Linux

cat /etc/passwd

低权限

对于拿到的内网主机是低权限的话

可以查看系统的版本信息以及补丁的更新信息，探测有没有一些nday，可以利用这些漏洞对主机进行提权的操作

高权限

如果拿到的机子是高权限或者进行提权之后，就可以考虑收集一些主机hash值等

Dump lsass内存

可以利用mimikatz这类工具进行内网主机NTLM hash的收集为之后的提权、持久化、委派提供良好的信息基础

计划任务

利用计划任务如at、schtasks可以进行计划任务的创建，这对权限的持久化有着巨大的帮助

创建后门

之后也可以创建后门，这也是为了进行权限维持，后门账户、后门木马等

域环境

是否有域环境

可以ipconfig /all这个命令来查询主机名以此判断该主机在内网环境的功能，以及判断是否存在域环境

下图显示就是该主机是处于一个域环境之中的，没有域环境就只会显示WORKGROUP工作组

定位域控

一般整个域内的时间都是由域控进行同步的，所以可以通过时间来判断域控主机的主机名

还可以直接net group “Domian controllers” /Domain来查询域控名

除此之外域控一般还用于充当DNS服务器，可以利用这个来定位域控

注意：Linux系统可以是域控制器的，但是功能会比较少，且难以管理，所以一般来说域控制器使用Windows系统来实现的，Linux用作服务器

进程

进程查看

杀软信息

规避敏感操作

如果该系统部署了杀软的话就需要注意避免敏感的操作

这些一系列行为几乎都会留下痕迹，日志等，再加上杀软的实时监测，很容易暴露自己的身份被溯源

免杀对抗

当获取杀软信息之后就可以做免杀对抗

针对不同的杀软进行针对性的免杀操作：360、火绒、Windows Defender等

载荷投递

用于对目标主机进行木马的下载

可以通过IPC$共享或者搭建SMB服务器，然后配合计划任务通过UNC路径来传入攻击载荷

系统程序下载：wget、curl、certutil、浏览器

webshell上传：蚁剑、冰蝎、哥斯拉

Windows白名单程序：mshta、msiexec

MSI是Windows中一个重要的服务，有个特性是以高权限来运行，生成.msi的攻击载荷可以达到提权的目的

脚本语言下载：asp、php、jsp这些类型的木马

shellcode：哥斯拉、shellcode加载器

进程路径信息

查找定位一些程序的位置，比如远控软件、微信、邮箱的安装路径等

DNS是否正常/TCP出否出网

判断C2上线的方式

TCP出网，DNS正常

就可以利用云函数进行上线

TCP不出网

可以考虑DNS ICMP

判断隧道的使用方式

TCP出网

可以利用反向TCP，如frp、nps的反向代理将数据外带

TCP不出网
正向HTTP

Neo-reGeorg

反向ICMP

frp+icmptunnel

进行反向的icmp隧道

代理服务器

如果该主机中TCP和ICMP均不出网，但是却配置了 代理服务器，就可以C2/隧道配置代理可出网

特定端口出网

就可以修改C2的监听器和隧道的服务器端口

本地端口开放情况

端口开放查询

通过端口开放情况就可以制定下一步的利用方法，比如开启了139、445端口就可以考虑是否可以利用IPC$共享来获取连接，实现攻击载荷的上传，进行进一步的横向

攻击脆弱服务

MSSQL执行命令

可以试试能不能绕过杀软进行添加用户以及文件上传的操作

web高权限服务
远程桌面服务
查询在线用户

有在线用户千万不要去挤人家，容易被发现，rdp的流量是十分敏感的，登进去别人就得退出来

查询端口
开启服务

net start可以查看已经开启的服务

关闭防火墙

hash免密登录

常用服务端口变化

SSH（22）：22022、2222

RDP（3389）：33888、33899

有些系统出于对安全性的考虑，会将一些常用的服务端口进行端口的改变使得不容易被攻击者发现

密码连接信息

SSH远程连接的工具：xshell、secureCRT、FinalShell等

数据库工具：Navicat

文件传输工具：winSCP

浏览器：IE、firefox、chrome、360

浏览器中一般会存有系统的密码，可以利用HackBRowserData导出浏览器中的密码、历史记录cookies以及书签等

https://github.com/moonD4rk/HackBrowserData

运行可得到results

然后打开就可以查看各个浏览器所保存的信息了

可以看到在浏览器下载的记录也可以获取到

RDP连接记录

注册表：有些注册表会明文存储密码

配置文件：配置文件中也会可能存在一些明文密码，配置不当造成或者忘记删除了

wifi：WiFi密码获取

SVN：可以获取一些项目的共享资源，如源代码、文档等

系统的账号密码

Windows
dump lsass内存

可以利用procdump+mimikatz配合使用，绕过杀软获取主机的密码

先利用procdump获取到lsass.dmp

然后利用mimikatz再进行密码的读取，即可获得该主机的密码

为什么能绕过杀软？

因为pecdump这个软件是由微软开发的一个工具，并不会被杀软识别为恶意的软件进行查杀

主要是将lsass.dmp外带到自己的攻击机上进行mimikatz读取的，所以达到了免杀的目的

SAM

位于C:\Windows/system32/config/

安全账号管理器，作用是对Windows账户安全进行管理，是通过“安全标识”实行的，是每个账户的唯一标识

修改注册表记录明文密码

在win2012R2及以上系统中，默认是在内存缓存中禁止保存着明文密码的。

但是攻击者可以通过修改注册表的方式抓取明文密码，但是这个更改之后是需要用户重新登录才可以成功抓取的

修改注册表信息

然后重启之后通过mimikatz进行明文密码的读取

Linux
strace SSHD进程/SSH程序

可以利用Strace监听SSH来源流量来记录密码后门

先需要查找sshd进程的PID

然后开始抓取流量

这个状态就是在监听流量了

然后任意一个用户登录或者ssh连接就可以记录到密码信息

我们另起一个终端用来模拟用户ssh连接

这边监听的终端就会有数据流入

然后流量信息就会保存在/tmp/.ssh.log中了，这样就可以在这里面寻找记录的明文密码信息了，由于/.tmp/.ssh.log中的内容比较多而杂，我这边的密码是123456所以就直接关键字查找了，正常得去翻看文件内容

. bash_history、.ssh/known_host文件

.bash_history文件记录了操作历史，可以读取这个文件查看管理员的操作历史以获取有价值的信息

.ssh/known_host文件是ssh的连接记录

键盘记录/截图
